# Structure Rules

Use these rules to validate article TSX files in this repository.

## Required sections

1. Definition & Context
2. Core Concepts
3. Architecture & Flow
4. Trade offs & Comparison
5. Best practices
6. Common Pitfalls
7. Real-world use cases
8. Common interview question with detailed answer
9. References

## Must also verify

- interview section is second-last
- references section is last
- no code examples or `<pre>` tags remain
- no `Implementation Example` or `Example code` sections remain
- `ArticleImage` is used for article SVGs where relevant

## Report shape

Use:

```text
Article:
Pass/Fail:
Violations:
Missing sections:
Order issues:
Suggested fixes:
```
