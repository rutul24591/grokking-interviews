# Article Checklist

Use this checklist before considering the article draft complete.

## Structure

- `ArticlePage` component name
- `<section>` per content block
- Correct heading hierarchy
- Required sections in the locked order
- Interview questions second-last
- References last

## Content

- No code examples
- No `<pre>` tags
- Paragraph-heavy writing style
- Production-scale trade-offs included
- 4-6 detailed interview questions and answers
- References included at the end
- Topic-specific language rather than generic system design filler
- Clear explanation of failure modes and operational trade-offs

## Visuals

- Exactly 3-5 SVG diagrams planned
- `ArticleImage` used for diagrams
- Captions explain what the diagram teaches

## Red flags

- repeated paragraphs with minor wording changes
- bulleted notes standing in for deep explanation
- generic interview questions that would fit any topic
- references that are too sparse or clearly low quality
- architecture diagrams that do not map to the article's actual argument

## Repo alignment

- Topic confirmed from `concepts/hierarchy-data.txt`
- Slug and file path aligned with sibling articles
- Style aligned with the reference articles listed in `AGENTS.md`
