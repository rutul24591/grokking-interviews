# Enhancement Checklist

Use this checklist before treating an enhanced article as complete.

## Structure

- Required sections are present
- Required sections are in order
- Interview section is second-last
- References section is last
- `ArticlePage` still reads cleanly as a TSX article component

## Content

- Weak sections are materially improved rather than lightly reworded
- Repetition has been reduced
- Paragraph-heavy style has improved
- Trade-offs and production failure modes are clearer
- Real-world examples are more concrete
- Interview questions are detailed and topic-specific

## Visuals

- Existing SVG usage still makes sense
- Missing diagrams are identified
- Placeholder visual thinking has been replaced with a concrete SVG plan where possible

## Cleanup

- No code blocks
- No `<pre>` tags
- No `Implementation Example` or `Example code` sections
- References are credible and sufficiently technical

## Red flags

- the edit reads like a rewrite with no respect for good existing material
- the article still feels generic after enhancement
- references remain sparse
- SVGs are mentioned but not clearly planned
