# Example 1 — Semantic page layout (end-to-end)

Demonstrates:

- semantic landmarks (`<header>`, `<nav>`, `<main>`, `<footer>`) to reduce boilerplate noise
- content boundaries (`<article>`, `<section>`, `<aside>`) to make “main content” obvious to crawlers
- a crawl-friendly, semantic article index (lists + per-item `<article>`)
- practical details: skip link, breadcrumbs, proper `<time>` usage
