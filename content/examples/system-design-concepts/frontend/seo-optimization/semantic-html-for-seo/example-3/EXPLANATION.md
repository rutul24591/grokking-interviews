# Example 3 — Semantic patterns for “non-article” content (advanced)

Demonstrates:

- `<dl>` for attribute/value data (specs, pricing, dimensions)
- `<figure>` + `<figcaption>` for meaningful media descriptions
- `<details>` / `<summary>` for FAQ content (crawlable + accessible)
- multiple `<nav>` landmarks with clear labels (primary vs breadcrumbs vs in-page)
