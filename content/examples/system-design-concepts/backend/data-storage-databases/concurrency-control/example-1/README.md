# Concurrency Control Example

This example simulates optimistic vs pessimistic locking using an in-memory record.

## Files
- `record-store.js`: Record with versioning.
- `optimistic.js`: Optimistic update workflow.
- `pessimistic.js`: Pessimistic lock workflow.
- `demo.js`: Runs both approaches.

## Run
`node content/examples/backend/data-storage-databases/concurrency-control/example-1/demo.js`
